# 03.2-the-api-key-gate

## Page Metadata

| Property | Value |
|---|---|

| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To ensure a user has an API key before they can manage subscriptions.

**Entry Context:** Alex has clicked the "Manage Subscriptions" button from the empty feed page (scenario 03.1), and he does not yet have an API key saved in the system.

**Exit Action:** Alex successfully adds an API key and is redirected to the "Manage Subscriptions" page.

## User Situation

Following the prompt on the empty feed page, Alex has decided to add a subscription. He is now entering the setup flow.

## Mental State

-   **Engaged:** He is actively following the application's guidance.
-   **Task-focused:** His goal is to get to the subscription management screen.

## On-Page Interactions (MVP)

1.  After clicking "Manage Subscriptions", Alex is redirected to the `/settings/api-keys` page.
2.  A message at the top of the page explains: "You need to add an API key before you can subscribe to new accounts."
3.  Alex follows the flow described in scenario `04.1-manage-api-keys` to add his Gemini key using the in-table add form.
4.  Upon successful submission of the key, the system automatically redirects him to the `/settings/subscriptions` page.

## Post-MVP Enhancements

-   In a post-MVP release, this page will include a "Skip for now" button.
-   Clicking "Skip" would allow the user to proceed to the `/settings/subscriptions` page but they would only be able to subscribe to a limited number of "shared" accounts that the platform pays for. This text would be explained next to the button.
