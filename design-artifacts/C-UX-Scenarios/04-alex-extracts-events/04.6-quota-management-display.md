# 04.6-quota-management-display

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 04-Alex-Extracts-Events |
| **Page Number** | 04.6 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To display the user's remaining API quota on the manual post selection screen.

**Entry Context:** A user is on the manual post selection screen.

**Exit Action:** The user is aware of their remaining quota and makes a selection.

## User Situation

A user, "Isabelle", is on the manual post selection screen and wants to know how many more posts she can select for extraction.

## Mental State

-   **Cautious:** Isabelle wants to avoid exceeding her API quota.
-   **Informed:** She wants to make an informed decision about which posts to extract.

## On-Page Interactions

1.  Isabelle is on the manual post selection screen.
2.  A summary bar is displayed, showing "Selected Posts: 0 / 50" (assuming her quota is 50).
3.  As she selects posts, the summary bar updates in real-time (e.g., "Selected Posts: 3 / 50").
4.  If she tries to select more posts than her remaining quota allows, the checkbox for the additional post is disabled, and a tooltip appears on hover, saying "You have reached your quota limit."
