# 07.1-capacity-limit-notification

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 07-Api-Management |
| **Page Number** | 07.1 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To inform the user when they cannot add more subscriptions due to capacity limits.

**Entry Context:** A user attempts to add a new subscription when the MVP's capacity limit for new social media account subscriptions is reached.

**Exit Action:** The user acknowledges the message and returns to the previous page.

## User Situation

A user, "David", is trying to add a new social media subscription, but the system has reached its capacity limit.

## Mental State

-   **Confused:** David doesn't understand why he can't add a new subscription.
-   **Frustrated:** He wants to use the feature but is being blocked.

## On-Page Interactions

1.  David clicks the "Add Subscription" button.
2.  The system checks the current number of subscriptions and determines that the capacity limit has been reached.
3.  An in-app message appears, stating: "We're sorry, but we've reached our current capacity for new subscriptions. We're working on expanding our capacity and will notify you when you can add more."
4.  The message includes an "OK" button to dismiss it.
5.  David clicks "OK" and is returned to the "Manage Subscriptions" page.
