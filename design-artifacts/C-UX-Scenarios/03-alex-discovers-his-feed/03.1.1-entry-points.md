## Overview

**Page Purpose:** To describe the different entry points for the subscription and event extraction flows.

## Entry Points

### 1. From Empty Feed
- **Context:** Alex is on his feed page, but it's empty because he has no subscriptions.
- **Action:** He clicks the "Manage Subscriptions" button (as described in scenario 03.1).
- **Flow:**
    - The application initiates the wizard.
    - The wizard steps are: "Add API Key", "Manage Subscriptions", "Extract Events".
    - The `exitPath` for the wizard is the feed page.

### 2. From User Menu
- **Context:** Alex clicks on the "Extract event from post(s)" menu item in the user menu.
- **Flow:**
    - The system checks if the user has API keys and subscriptions.
    - **If both exist:** The user is navigated directly to the `/posts/extract` page.
    - **If either is missing:** The application initiates the wizard.
        - The `currentStep` of the wizard is set to the first missing prerequisite (API key or subscriptions).
        - The wizard will include all subsequent steps, including "Extract Events".
        - The `exitPath` for the wizard is the feed page.
