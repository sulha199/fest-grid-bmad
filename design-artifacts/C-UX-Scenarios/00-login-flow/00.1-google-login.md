# 00.1-google-login

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 00-Login-Flow |
| **Page Number** | 00.1 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To handle user authentication via Google Login, ensuring the user is logged in before proceeding to features that require authentication, such as providing an API key.

**Entry Context:** User attempts to access any feature that requires an API key (e.g., "Add Subscription", "Extract event from post(s)"), but is not currently logged in.

**Exit Action:** User successfully logs in with Google and is redirected to the API-key gate.

## User Situation

A new user, "Chloe", wants to use a feature that requires an API key.

## Mental State

-   **Goal-oriented:** Chloe wants to access a specific feature.
-   **Expectant:** She expects a smooth and familiar login process.

## On-Page Interactions

1.  Chloe attempts to access a feature that requires an API key.
2.  The system checks if an API key is present. It is not.
3.  The system then checks if Chloe is authenticated. She is not.
4.  A modal or a redirect to a login page appears, presenting a prominent "Sign in with Google" button.
5.  Chloe clicks the "Sign in with Google" button.
6.  The standard Google login pop-up appears.
7.  Chloe selects her Google account and enters her credentials if prompted.
8.  After successful authentication with Google, the pop-up closes.
9.  Chloe is now logged into FestGrid.
10. The system redirects Chloe to the API-key gate, so she can provide her key and then proceed to the feature she originally intended to access.
