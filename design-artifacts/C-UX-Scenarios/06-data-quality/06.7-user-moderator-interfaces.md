# 06.7-user-moderator-interfaces

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 06-Data-Quality |
| **Page Number** | 06.7 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To provide interfaces for users to view their submitted reports and for moderators to manage them.

**Entry Context:** A user or a moderator navigates to their respective reports page.

**Exit Action:** The user or moderator has finished reviewing the reports.

## User Situation

**User:** A user, "Grace", wants to check the status of a report she submitted earlier.
**Moderator:** A moderator, "Henry", needs to review and take action on pending user reports.

## Mental State

-   **Grace (User):** Curious about the status of her report.
-   **Henry (Moderator):** Focused on maintaining the quality of the platform.

## On-Page Interactions

### User Interface

1.  Grace clicks on her user menu and selects "My Reports".
2.  The "My Reports" page loads, displaying a list of all the reports she has submitted.
3.  Each report in the list shows the event name, the reason for the report, the date submitted, and the current status (e.g., "Pending", "Resolved").

### Moderator Interface

1.  Henry clicks on his user menu and selects "Moderator Items".
2.  The "Moderator Items" page loads, displaying a list of all user reports that require action.
3.  Henry can filter the reports by type (e.g., "Cancelled", "Dangerous") and status.
4.  He clicks on a report to view the details.
5.  After reviewing the report, he can take action, such as "Soft-delete event", "Mark as safe", or "Dismiss report".
