# 03.3-adding-a-subscription

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 03-Alex-Discovers-His-Feed |
| **Page Number** | 03.3 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To manage (add, view, delete) social media account subscriptions.

**Entry Context:** Alex has successfully added an API key and is now on the "Manage Subscriptions" step of the wizard (scenario 03.2).

**Exit Action:** Alex has a new subscription in his list and clicks the "Complete" button in the wizard to return to his feed.

## User Situation

Having added his API key, Alex is now ready to subscribe to his favorite band's social media account.

## Mental State

-   **Ready to complete the task:** He is on the final step of the process.
-   **Expects a simple table interaction:** He has seen this UI pattern before on the API key page.

## On-Page Interactions

### Viewing Subscriptions
- Alex lands on the "Manage Subscriptions" page.
- He sees a table designed to list his subscriptions. The columns are "Platform", "Account ID / URL", and "Actions".
- The table is initially empty. The last row is an editable "Add New" form.

### Adding a New Subscription
- Alex focuses on the "Add New" row.
- He selects "Twitter/X" from the "Platform" dropdown.
- In the "Account ID / URL" input, he starts typing the band's name, "The Weekend Warriors". The input shows an autocomplete suggestion for an existing shared account with that name. He selects it.
- He clicks the "Add" (`+`) button in the "Actions" column of that row.
- The new subscription for "The Weekend Warriors" is added to the table above the "Add New" row.
- The "Add New" row clears itself, ready for another entry.
- A success toast notification appears: "Subscription added."

### Navigating Back
- Satisfied, Alex clicks the "Complete" button in the wizard.
- He is redirected to the `exitPath` that was configured at the start of the wizard flow (his feed).
