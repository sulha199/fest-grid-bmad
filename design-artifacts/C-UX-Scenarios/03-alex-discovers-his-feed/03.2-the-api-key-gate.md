# 03.2-the-api-key-gate

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 03-Alex-Discovers-His-Feed |
| **Page Number** | 03.2 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To ensure a user has an API key before they can manage subscriptions.

**Entry Context:** Alex has clicked the "Manage Subscriptions" button from the empty feed page (scenario 03.1). The current path is recorded as the `returnPath`.

**Exit Action:** Alex successfully adds an API key and is redirected to the `returnPath`.

## User Situation

Following the prompt on the empty feed page, Alex has decided to add a subscription. He is now entering the setup flow.

## Mental State

-   **Engaged:** He is actively following the application's guidance.
-   **Task-focused:** His goal is to get to the subscription management screen.

## On-Page Interactions (MVP)

This flow describes how a user is guided to add an API key and manage subscriptions using the wizard page.

1.  **Trigger:** User clicks "Manage Subscriptions". The application records the current path as the `returnPath`.
2.  **Action:** The application constructs the wizard configuration:
    -   `steps` array:
        1.  `path`: `/settings/api-keys`, `title`: "Add API Key", `canSkipStep`: `!isMvp`
        2.  `path`: `/settings/subscriptions`, `title`: "Manage Subscriptions"
    -   `exitPath`: the `returnPath`.
    -   `currentStep`: If the user has at least one API key, `currentStep` is `1`, otherwise it is `0`.
3.  **Redirect:** The application redirects the user to the `/wizard` page with the constructed query parameters.
4.  **User Interaction (starting from API key step):
    -   Alex is shown the `/settings/api-keys` page within the wizard.
    -   He can either add an API key or click "Skip Step" (if `!isMvp`).
    -   If he adds a key, `isStepCompleted` is set to `true`, and he clicks "Next Step".
    -   He is now on the `/settings/subscriptions` step.
    -   He manages his subscriptions.
    -   He clicks "Complete" and is redirected to the `returnPath`.
5.  **User Interaction (starting from subscriptions step):
    -   Alex is shown the `/settings/subscriptions` page within the wizard.
    -   He manages his subscriptions.
    -   He clicks "Complete" and is redirected to the `returnPath`.

## Post-MVP Enhancements

-   When `isMvp` is `false`, the `canSkipStep` property will be true, allowing the user to skip the API key step. If skipped, the user can only subscribe to a limited number of "shared" accounts on the subscriptions page.
