# 05.1-extract-event-from-post

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 05-Alex-Extracts-Events |
| **Page Number** | 05.1 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To allow users to manually select social media posts for event extraction.

**Entry Context:** Alex clicks on the "Extract event from post(s)" menu item under the user menu.

**Exit Action:** Alex selects posts and initiates the extraction process.

## User Situation

Alex has already subscribed to several social media accounts and now wants to manually select specific posts to be processed for events.

## Mental State

-   **Focused:** He has a clear goal to extract events from specific posts he has seen.
-   **In Control:** He appreciates the ability to manually select posts, giving him more control over API quota usage and the relevance of extracted events.

## On-Page Interactions

### Entry Logic
1.  **User Action:** Alex clicks the "Extract event from post(s)" menu item.
2.  **System Check:** The system checks for the existence of API keys and subscribed accounts.
    -   **If no API key:** The user is redirected to the API key wizard (`/wizard` with the API key step active). The `exitPath` for the wizard is set to the current page (`/posts/select`).
    -   **If no subscriptions:** The user is redirected to the subscription management step of the wizard (`/wizard` with the subscription step active). The `exitPath` is set to `/posts/select`.
    -   **If both exist:** The user is taken to the "Manual Post Selection" page (described in scenario 03.5).
