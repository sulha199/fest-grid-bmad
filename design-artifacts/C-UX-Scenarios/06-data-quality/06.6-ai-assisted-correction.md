# 06.6-ai-assisted-correction

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 06-Data-Quality |
| **Page Number** | 06.6 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To allow users with an API key to pre-fill the event correction form using a URL to a social media post.

**Entry Context:** A user with a BYOK key is on the event correction form and wants to use the AI-assisted correction feature.

**Exit Action:** The user reviews and approves the pre-filled correction form.

## User Situation

A user, "Frank", has his own Gemini API key and wants to correct an event's details. He has a URL to a social media post with the correct information.

## Mental State

-   **Efficient:** Frank wants to correct the information as quickly and easily as possible.
-   **Tech-savvy:** He is comfortable using an API key and an AI-powered feature.

## On-Page Interactions

1.  Frank is on the event correction form.
2.  He clicks on the "AI-Assisted Correction" button.
3.  A text input field appears, prompting him to enter a URL.
4.  Frank pastes the URL to the social media post and clicks "Extract".
5.  The system uses his API key to call the AI agent and extract the event information from the URL.
6.  The correction form is pre-filled with the extracted information.
7.  Frank reviews the pre-filled data, makes any necessary adjustments, and clicks "Submit".
8.  The correction is submitted.
