# 04.1-manage-api-keys

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 04-Alex-Manages-Keys |
| **Page Number** | 04.1 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To allow users to add, view, and delete their personal API keys for services like Gemini.

**Entry Context:** Alex navigates to the "Manage API Keys" page from his user profile menu.

**Exit Action:** Alex returns to his profile page after managing his keys.

## User Situation

Alex wants to use the social media subscription feature, and he knows he needs to add his Gemini API key to FestGrid.

## Mental State

-   **Task-oriented:** He has a specific goal to add his API key.
-   **Security-conscious:** He expects the application to handle his API key securely (e.g., by masking it).

## On-Page Interactions

### Viewing API Keys
- Alex lands on the "Manage API Keys" page.
- He sees a table designed to list his keys. The columns are "Provider", "API Key", and "Actions".
- The table is initially empty, with a disabled "Add New" row at the bottom.

### Adding a New API Key
- The "Add New" row has a dropdown for "Provider" (defaulting to "Gemini"), a text input for the API key, and an "Add" button.
- A small link next to the API key input says "How to get a Gemini API key?", which links to a help document.
- Alex selects "Gemini" (it's already selected).
- He pastes his key into the input field.
- He clicks the "Add" button.
- The new key appears in the table above. The key is masked, showing only the first 5 characters (e.g., `sk-Abc12...`).
- A success toast notification appears: "API Key saved."

### Deleting an API Key
- Alex decides to delete an old, invalid key he added previously.
- He clicks the "Delete" button in the row for the old key.
- The row does not disappear. Instead, its content becomes greyed out, and the "Delete" button is replaced with an "Undo" button.
- A toast notification appears: "API Key deleted." with an "Undo" action.
- Alex ignores the undo option and navigates to another page. When he returns to the "Manage API Keys" page later, the old key is gone.
