# 06.5-data-inconsistency-checks

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 06-Data-Quality |
| **Page Number** | 06.5 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To prevent users from submitting inconsistent data when correcting event information.

**Entry Context:** A user is submitting a correction for an event and enters inconsistent data (e.g., end date before start date).

**Exit Action:** The user corrects the inconsistent data and successfully submits the form.

## User Situation

A user, "Emily", is correcting the details of an event and accidentally enters an end date that is before the start date.

## Mental State

-   **Focused:** Emily is trying to correct the event details.
-   **Potentially Rushed:** She may be trying to complete the form quickly and makes a mistake.

## On-Page Interactions

1.  Emily is on the event correction form.
2.  She enters a start date of "2026-08-15" and an end date of "2026-08-10".
3.  She clicks the "Submit" button.
4.  The system validates the form data and detects the inconsistency.
5.  An inline error message appears next to the end date field, stating: "End date cannot be earlier than the start date."
6.  The form is not submitted.
7.  Emily corrects the end date to "2026-08-16".
8.  She clicks "Submit" again.
9.  The form is successfully submitted.
