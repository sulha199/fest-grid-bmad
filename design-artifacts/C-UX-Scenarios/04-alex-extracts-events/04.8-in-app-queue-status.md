# 04.8-in-app-queue-status

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 04-Alex-Extracts-Events |

| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To provide users with a real-time view of the queue status for their subscribed posts.

**Entry Context:** A user navigates to the "Queue Status" section in their user menu.

**Exit Action:** The user has reviewed the queue status and is informed about the performance of their API keys.

## User Situation

A user, "Paul", wants to check the status of his subscribed posts and see if there are any issues with his API keys.

## Mental State

-   **Curious:** Paul wants to understand how his subscriptions are being processed.
-   **Proactive:** He wants to identify and resolve any potential issues with his API keys.

## On-Page Interactions

1.  Paul clicks on his user menu and selects "Queue Status".
2.  The "Queue Status" page loads, displaying a list of his subscriptions.
3.  For each subscription, the page shows:
    -   The name of the social media account.
    -   The number of posts currently in the queue.
    -   The estimated time for the posts to be processed.
    -   The status of the API key being used for that subscription (e.g., "Active", "Rate-limited", "Invalid").
4.  If there are any issues (e.g., a large queue, an invalid key), a clear warning message is displayed with a link to resolve the issue.
