# 03.5-manual-post-selection

## Page Metadata

| Property | Value |
|---|---|
| **Scenario** | 03-Alex-Discovers-His-Feed |
| **Page Number** | 03.5 |
| **Platform** | Web (Desktop) |

## Overview

**Page Purpose:** To allow users to manually select social media posts for event extraction. This page is a new step in the subscription wizard.

**Entry Context:** Alex has just added a new subscription (scenario 03.3) and clicks "Next Step" in the wizard.

**Exit Action:** Alex selects the posts he is interested in and clicks "Extract Events".

## User Situation

Alex has just subscribed to a few accounts and is now presented with a screen to choose which posts to process.

## Mental State

-   **Curious:** He is interested to see the posts from the accounts he just subscribed to.
-   **Engaged:** He is in an interactive process and is willing to make selections.

## On-Page Interactions

### Main UI
- The UI consists of a tab view at the top, a summary bar below it, and the content area.
- A primary button "Extract Events" is at the bottom right of the page.

### Tab View
- Each subscribed account is represented as a tab. The tab title is the `displayName` of the `SocialMediaAccountProfile`.
- If an account is newly added (`isNewlyAdded` is `true`), its tab is automatically activated.
- **Inactive Account Warning:** If an account has not had any posts in the last 30 days, an alert icon is shown next to the tab title.

### Summary Bar
- This bar displays:
    -   "Selected Posts: X / Y" where X is the number of posts selected and Y is the user's remaining API quota.
    -   The text is colored red if X > Y.

### Content Area
- The content area displays the content of the active tab.

#### Content for Active Tab (with recent posts)
- A list of the latest 20 posts from the selected social media account is displayed in a card-based layout.
- Each card represents a post and contains:
    -   A checkbox at the top right corner.
    -   The post's content (text and image).
- **Selection:** Alex can click the checkbox on each card to select or deselect a post.
- The selected state of posts is preserved when Alex switches between tabs.

#### Content for Inactive Tab
- If the active tab is for an account with no posts in the last 30 days, the content area shows:
    -   A warning message: "This account has not posted in the last 30 days."
    -   A "Remove Subscription" button.
    -   Clicking the button initiates the soft-delete process. The tab is greyed out and an "Undo" button appears.

### Extraction Process
- When Alex clicks the "Extract Events" button, the system validates that the number of selected posts does not exceed the remaining quota.
- If the validation passes, the selected posts are sent to the AI agent for processing.
- After the extraction is initiated, Alex is redirected to the wizard's `exitPath` (his feed).
