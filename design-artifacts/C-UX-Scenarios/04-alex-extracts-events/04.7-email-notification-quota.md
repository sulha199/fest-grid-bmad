# 04.7-email-notification-quota

## Page Metadata

| Property | Value |
|---|---|

## Overview

**Page Purpose:** To notify users via email when their subscribed posts are queued due to Gemini API quota exhaustion.

**Entry Context:** The system detects that a user's subscribed posts have been in the queue for a specified number of days due to quota issues.

**Exit Action:** The user is informed about the issue and encouraged to contribute an additional API key.

## User Situation

A user, "Maria", has subscribed to several social media accounts, but her API key has run out of quota. As a result, her posts are stuck in the processing queue.

## Mental State

-   **Unaware:** Maria may not be aware that her posts are not being processed.
-   **Potentially Confused:** She might wonder why she is not seeing new events from her subscriptions.

## On-Page Interactions

1.  The system identifies that 3 of Maria's subscribed posts have been in the queue for 3 days.
2.  An automated email is sent to Maria's registered email address.
3.  The email has a clear subject line, such as "Action Required: Your FestGrid event extraction is paused."
4.  The body of the email explains that some of her subscribed posts are not being processed due to API quota exhaustion.
5.  The email suggests that she can resolve this by contributing an additional API key and provides a link to the API key management page.
